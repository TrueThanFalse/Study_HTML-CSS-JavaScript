정규식(Regular Expression) : 문자열에서 특정 내용을 찾거나 대체, 발췌하는데 사용

  • regexp 단어로 많이 사용됨
  • 특정 데이터를 탐색, 확인, 존재여부 등을 쉽게 확인하기 위한 규칙
  • 웹 분야에서는 password 작성규칙, 첨부파일 규칙 등에 사용되며 자주 쓰이진 않음
  • 데이터 분석과 관련해서는 반드시 꼭 알고 있어야 함
  • 알고 있으면 편리한 기능이 많음. 다른 언어에서도 거의 비슷한 형식으로 활용됨
  • /regexp/i : (/ /) = 시작과 종료 기호, regexp = pattern, i = flag 라고 명명함
  • ("str").match(/regexp/flag) : str 에서 매칭되는 항목들을 배열로 return
  • ("str").replace(/regexp/flag, 변환할 문자) : 정규표현식에 매칭되는 항목을 변환할 문자로 변경
  • ("str").split(/regexp/) : 정규표현식에 매칭되는 항목으로 쪼개어 배열로 return
  • (정규표현식).test("str") : str 문자열이 정규표현식 규칙에 맞으면 true, 아니면 false
  • 정규표현식 flag
  • i : 대소문자 구분 X
  • g : 전역 검색
  • m : 줄바꿈도 포함
  • 정규표현식 pattern
  • a-zA-z : 모든 영어 알파벳, a-z : 소문자만, A-Z : 대문자만
    ㄱ-ㅎ가-힣 : 모든 한글 / 0-9 : 숫자
  • . : 모든 문자열(숫자, 한글, 영어, 특수기호, 공백 모두), 단 줄바꿈은 제외
  • \d : 숫자 \D : 숫자 아닌 것
    \s : 공백 \S : 공백 아닌 것
    \w : 영어+숫자 \W : 영어+숫자 아닌 것
    \특수기호 : \.\!\@\#...
  • 검색 패턴 : | => or의 의미, e.g. a|b
  • [] : 괄호 안의 문자들 중 하나, or 처리의 묶음으로 보면 됨 /abc/ => "abc"를 포함하는 /[abc]/ => "a" 또는 "b" 또는 "c" 를 포함하는 [다-바] => 다,라,마,바 중 하나
  • [^문자] : 괄호 안의 문자를 제외한 것 [^abcd] : a,b,c,d 4개의 문자가 아닌 것
  • ^문자열$ : 문자열로 시작(^)하는, 문자열로 끝($)나는
